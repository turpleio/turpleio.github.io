왜 터플인가? Part 5 - 가볍고 유연한 구조

March 5, 2023 · 2 min read

Turple Team

터플은 엣지 클라우드와 같이 작은 규모의 서비스를 위해서 가벼운 도커 클러스터를 구성할 수도 있고, 빈번한 스케일링이 요구되는 큰 규모의 서비스를 위해서 쿠버네티스 클러스터를 사용할 수도 있습니다.

클러스터 구조 예제#

아래 예제들은 터플을 사용하여 클라우드 네이티브 환경을 구축하기 위한 최소 구성부터 다양한 용도에 맞는 구조를 보여주고 있습니다.

구조 1 - Minimum#

  • 최소 HW 구성
  • 고가용성
  • CI/CD

구조 2 - Standard#

  • 고가용성
  • CI/CD
  • Redis
  • Mongodb
  • 모니터링

구조 3 - Large#

  • 스케일링 가능한 워커 클러스터
  • 용도별 클러스터 구성
  • 고가용성
  • CI/CD
  • Redis
  • Mongodb
  • 모니터링
  • 로깅

구조 4 - Hybrid (Docker + Kubernetes)#

  • 쿠버네티스 클러스터
  • 고가용성
  • CI/CD
  • Redis
  • Mongodb
  • 모니터링
