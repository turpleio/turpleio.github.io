터플의 관리 영역

인프라생성부터 서비스운영까지 클라우드환경의 모든 영역을 손쉽게 관리할 수 있습니다.

지원 클라우드 서비스

TURPLE CLI

사용자 친화적인 CLI를 사용하여 클라우드에 구축된 운영환경을 손쉽게 관리할 수 있습니다.

$ tpcli create world
-- Create World
$ tpcli create world
$ tpcli create cluster dev-app --type app
-- Create Cluster
$ tpcli create cluster dev-app --type app
$ tpcli create node dev-app4 --type app
-- Create Node
$ tpcli create node dev-app4 --type app
$ tpcli app build hello --target dev-app
-- Build app
$ tpcli app build hello --target dev-app

Create World

운영환경 구조에 맞게 VM을 생성하고 CI/CD를 구축합니다.

Create Cluster

기존 운영환경에 3개의 노드로 구성된 새로운 클러스터를 추가합니다.

Create Node

기존 클러스터에 새로운 노드를 추가합니다.

Build App

지정된 클러스터에 어플리케이션을 빌드하여 배포합니다.

왜 터플인가?

빠른 클라우드 기술 도입

터플은 클라우드 네이티브 환경을 구축하기 위해 필요한 기능을 모두 담고 있기 때문에 최소의 비용으로 단기간에 클라우드 네이티브 기술을 도입하려는 기업에게는 최고의 솔루션입니다.

인프라 관리부터 어플리케이션 관리까지

클라우드 인프라의 관리 뿐만 아니라 어플리케이션의 빌드와 배포 그리고 모니터링까지 서비스 운영에 필요한 모든 것을 관리할 수 있습니다.

멀티클라우드

클라우드 제공사(IaaS)의 의존성 없이 멀티 클라우드 환경에 서비스 구축이 가능하고 모든 서비스를 한 곳에서 통합 관리 할 수도 있습니다.

빠른 시스템 확장 (자동화)

인프라 관리와 어플리케이션의 빌드와 배포, 클러스터 추가 등 서비스 개발과 운영에 필요한 반복적인 작업들이 자동화되어 있기 때문에 간단한 명령어로 빠른 시스템의 확장이 가능합니다.

가볍고 유연한 구조

엣지 클라우드와 같이 작은 규모의 서비스를 위해서 가벼운 도커 클러스터를 구성할 수도 있고, 빈번한 스케일링이 요구되는 큰 규모의 서비스를 위해서 쿠버네티스 클러스터를 사용할 수도 있습니다.

시장검증

터플은 이미 수년 동안 금융 핀테크(마이데이터서비스), 의료유통(백신콜드체인), 자동차산업(모빌리티, 전기차배터리, V2X보안)등 다양한 산업분야에서 클라우드 네이티브 플랫폼으로 사용되고 있습니다.

터플 대시보드

WebUI를 사용하여 클라우드에 구축된 운영환경의 관리와 서비스들의 동작상태를 모니터링할 수 있습니다.
